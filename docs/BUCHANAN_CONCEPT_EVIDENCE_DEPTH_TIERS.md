# Buchanan Concept Evidence Depth Tiers

## Purpose

This document prevents the Buchanan platform from treating every concept as if it requires the full Body without Organs prototype path.

BDP-001J.0 records the scalable rule:

```text
Not every concept requires full-source treatment.
Every concept requires only the evidence needed for the authority level it is allowed to carry.
```

## Core Doctrine

Concept detail follows authority.

```text
show term exists
→ light record

show concept is mentioned
→ concept mention + passage

show relation between concepts
→ relation + evidence passage

say Buchanan argues something
→ exact Buchanan source + passage + citation

generate synthesis
→ multiple evidence-backed records + authority label
```

## Tier 1 — Anchor Concepts

Anchor concepts are load-bearing terms for the platform.

Examples:

```text
Body without Organs
assemblage
desire
schizoanalysis
deterritorialisation
strata
war machine
becoming
capitalism
```

Required evidence depth:

```text
canonical source
passage
citation
concept mention
primary Deleuze and Guattari evidence where applicable
Buchanan evidence before Buchanan-specific claims
readback
eventual interpretation only after evidence exists
relations only with evidence
```

## Tier 2 — Supporting Concepts

Supporting concepts help explain or connect anchor concepts.

Examples:

```text
organism
plane of consistency
affect
territory
coding
decoding
reterritorialisation
```

Required evidence depth:

```text
concept record
at least one cited passage when promoted
reviewed concept mention
reviewed relation when used structurally
no Buchanan-specific claim without exact Buchanan source passage
```

## Tier 3 — Contextual Terms

Contextual terms can remain light until they become important.

Examples:

```text
psychoanalysis
capital flows
modernity
literature
institution
clinic
```

Required evidence depth:

```text
keyword or alias
proposed concept status
optional relation hint
no interpretation
no Buchanan claim
promotion required before structural use
```

## Authority Rule

The system may scale concept detail up or down, but it must not scale authority beyond evidence.

```text
Low evidence → low authority.
High authority → high evidence requirement.
```

## Buchanan Claim Rule

No Buchanan-specific claim may be made without an exact Buchanan source passage.

A placeholder Buchanan source candidate is not enough.

An exact Buchanan publication, lecture, transcript, interview, or teaching source must be specified, reviewed, cited, and linked before Buchanan-specific interpretation begins.

## BDP-001J.0 Boundary

```text
No SQL migration.
No database mutation.
No new source candidate.
No canonical source.
No passage.
No citation.
No concept mention.
No concept relation.
No interpretation.
No generated Buchanan claim.
```
